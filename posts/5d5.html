ssh服务讲解

SSH简介：
  SSH是一种安全通道协议，主要用来实现字符界面的远程登录，远程复制等功能
  ssh协议对通信双方的数据传输进行了加密处理，其中包括用户登录时输入的用户口令
  与telnet等应用相比，ssh协议提供了更好的安全性
  OpenSSH：ssh协议的开源实现

配置openssh服务器端：
  ssh服务是”C/S”架构的，C代表客户端，S代表服务器端
  在安装Linux系统时openssh服务默认已经安装

ssh服务及配置文件：
  openssh-server：服务器端软件
  openssh-clients：客户端软件
要保证这两个包都安装才行
  启动：service sshd start

配置文件：
  /etc/ssh/sshd_config
  配置项：
Port 22：port代表监听的端口，默认22，可修改
ListenAddress 0.0.0.0：监听的IP地址，默认为所有，可修改
Protocol 2：代表的ssh的第二个版本
LoginGraceTime 2m：输入用户名后，2分钟不输入密码自动关闭
PermitRootLogin yes：是否允许root登录
MaxAuthTries 6：最大的认证次数，即输入密码次数
RSAAuthentication yes：启用RSA算法认证
PubkeyAuthentication yes：启用公钥认证
AuthorizedkeysFile：~/.ssh/authorized_keys公钥放置位置，在对应用户的家目录下面的.ssh隐藏目录中，名字为authorized_keys文件
PasswordAuthentication yes：密码认证
PermitEmptyPasswords no：是否允许空密码登录

限制可登录用户的配置：（allow白名单，deny黑名单）
AllowUsers user1 user2 user3…
DenyUsers
AllowGroups
DenyGroups

登录验证方法：
  ssh服务支持两种验证方式：密码验证、密钥对验证，可以只设置其中一种方式，也可以两种方式都启用
  密码验证：以服务器中本地系统用户的远程登录名称、密码进行验证，这种方式使用最为简便，但以客户机角度来看，正在连接的服务器有可能被假冒，从服务器角度来看，当遇到密码穷举（即暴力破解）攻击时，防御力能力比较弱
  密钥对验证：要求提供相匹配的密钥信息才能通过验证

使用ssh客户端程序
客户端组件：
命令程序：ssh、scp、sftp
图形工具：Xshell

ssh远程登录命令
  格式：ssh [user@]host [command]
      -p port：远程服务器监听的端口
  例如：ssh test@192.168.0.10
    使用test用户登录192.168.0.10这台服务器，注意：test用户是192.168.0.10这台服务器上的用户

scp命令：
scp [options] SRC… DEST/
两种方式：
  scp [options] [user@]host:/sourcefile /testdir
  scp [options] /sourcefile [user@]host:/testpath

常用选项：
  -C：压缩数据流
  -r：递归复制
  -p：保持原文件的修改时间、访问时间、权限
  -P PORT：指明远程主机监听的端口

远程复制工具

rsync命令：
基于ssh和rsh服务实现高效率的远程系统之间复制文件
使用安全的shell连接做为传输方式
  rsync -av /etc/passwd server1:/tmp复制目录和目录下文件
  rsync -av /etc/ server1:/tmp只复制目录下文件

比scp更快，只复制不同的文件
选项：
  -n模拟复制过程
  -v显示过程
  -r递归复制目录树
  -p保留权限
  -t保留时间戳
  -g保留组信息
  -o保留所有者信息

ssh客户端：
允许实现对远程系统经验证地加密安全访问
当用户远程连接ssh服务器时会复制ssh服务器/etc/ssh/ssh_host*key.pub文件中的公钥到客户机的~./ssh/know_hosts中，下次连接时，会比较两处是否有不同

基于key认证

基于key认证：
在服务器、客户端中构建密钥对验证ssh的基本过程
  第一步：创建密钥对：
    公钥文件：id_rsa.pub
    私钥文件：id_rsa
  第二步：上传公钥文件id_rsa.pub到服务器中
  第三步：导入公钥信息
    公钥库文件：~/.ssh/authorized_key（此文件的权限必须为600）
  第四步：使用密钥对方式验证
具体实现两种方式：
第一种方式（较复杂）：
  使用user用户连接ssh服务器的user1用户
  第一步：在客户端创建密钥对
  su - 客户端用户
  ssh-keygen -t rsa -b 2048（一路回车）
  第二步：上传公钥
  cd /home/客户端用户/.ssh
  scp id_rsa.pub root@服务端IP:/tmp
  第三步：在ssh服务器中设置
  cd /tmp
  mkdir /home/服务端用户/.ssh
  cat id_rsa.pub > /home/服务端用户/.ssh/authorized_keys（相当于mv id_rsa.pub /home/服务端/.ssh/authorized_keys）
  chmod 600 /home/服务端用户/.ssh/authorized_keys
  第四步：在客户端登录验证

第二种方式（简单）
  使用客户端用户连接ssh服务器的服务端用户
  第一步：在客户机创建密钥对
  su - user
  ssh-keygen -t rsa -b 2048
  第二步：直接导入公钥到user1用户的家目录中
  ssh-copy-id -i id_rsa.pub user1@192.168.0.10（cd 到用户家目录下的.ssh目录下）
  输入user1用户的密码
  查看是否成功
  第三步：在客户端验证    
  ssh user1@192.168.0.10以user1用户登录看是否免密

ssh服务最佳实践
  1、不要使用默认端口
  2、禁止使用Protocol version 1
  3、限制可登录用户
  4、设定空闲会话超时时长
  5、利用防火墙设置ssh访问策略
  6、仅监听特定的IP地址
  7、基于口令认证时，使用强密码策略
  8、使用基于密钥的认证
  9、禁止使用空密码
  10、禁止root用户直接登录
  11、限制ssh的访问频度和并发在线数
  12、做好日志，经常分析

sudo提升权限

sudo的用途：

  1. sudo 能够授权指定用户在指定主机上运行某些命令，如果未授权用户尝试使用，sudo会提示联系管理员
  2. sudo 可以提供日志，记录每个用户使用sudo操作
  3. sudo 为系统管理员提供配置文件，允许系统管理员集中地管理用户的使用权限的使用的主机
  4. sudo 使用时间戳文件来完成类似”检票”的系统 默认存活期为5分钟的”入场券”
    通过visudo命令编辑配置文件，具有文件，具有语法检查功能
      visudo -c检查语法

配置sudo授权
配置文件：/etc/sudoers，/etc/sudoers.d/
日志文件：/var/log/secure

配置文件支持使用通配符glob：
  ?匹配单一字符
  *匹配任意长度字符
  [wxc]匹配其中一个字符
  [^wxc]除了给定的字符
  \x转义
  [[:alpha:]]字母 示例：/bin/ls [[:alpha:]]*
  !排除

使用方法：
  sudo [参数] [授权命令]
  参数：
  -u user：以指定用户身份运行命令
  -l：查看当前用户可以执行的命令
  -v：更新用户的时间戳而不执行命令
  -V：显示sudo的版本信息

配置文件格式：
使用visudo命令或执行vi /etc/sudoers打开配置文件
  文件中记录格式：
    用户 主机名列表=（代表用户）命令程序列表
    ALL： 代表所有
    NOPASSWD：免密
  用户：运行命令者的身份
  主机列表：通过哪些主机
  代表用户：以哪个用户的身份
  命令程序列表：运行哪些命令
  注意：如果只授权用户登录本机执行命令，主机写本地ip即可
例如：
  给user这个用户sudo权限，允许他在test这台主机中可执行/bin/下的所有命令，除了/sbin/reboot/外
    visudo
    user test=/sbin/*,!/sbin/reboot

现在user用户就可以使用授权的命令了
  su - user
  sudo授权命令（会提示输入密码，输入user用户的密码即可，输入密码后5分钟内不用再输入密码）

sudo别名
别名设置：
  别名有四种类型：
    User_Alias用户别名
    Runas_Alias代表用户别名（可以省略）
    Host_Alias主机别名
    Cmnd_Alias命令别名
（U、H、C后面可以跟多个用 ，分开） 

User_Alias U=haha
Host_Alias H=test
Cmnd_Alias C=/sbin/*
U H=C

格式：Alias_Type NAME1=item1,item2,item3…
例如：定义用户别名为USER，包括user1，user2，user3，user4四个用户
  User_Alias USER=user1,user2,user3,user4
这条别名就可以用来引用了
  USER test=/sbin/*

sudo别名
也可以将用户加入到组里进行统一管理，例如wheel组，组名前要加% 将用户加入到wheel组中，然后对wheel进行授权
%wheel test=ALL 

User_Alias U=haha
Host_Alias H=test
Cmnd_Alias C=/sbin/*
U H=C
%wheel H=ALL

这样wheel组里的所有用户都可以执行test主机里面的所有命令了

ssh端口转发

转发到远端：ssh -C -f -N -g -L 本地端口:目标IP:目标端口 用户名@目标IP
转发到本地：ssh -C -f -N -g –R 本地端口:目标IP:目标端口 用户名@目标IP

本地ssh不是22，需要最后指定本地端口
ssh -C -f -N -g -L 本机端口:远端IP:远端端口 root@远端IP -p port
