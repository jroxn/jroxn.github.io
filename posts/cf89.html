docker 报错 mongo

主要是介绍运维经验

因为运维的时候不知道是谁搭建的环境，更不知到安装的软件方式是什么样的。

故记录此次排错过程。

从上面日志中提取关键词 “timeout=30000” “192.168.1.113:8081” “拒绝连接” “mongodb-driver-core-3.8.2.jar!”

初步怀疑mongodb挂掉，IP端口为192.168.1.113:8081

登录主机，查询历史命令发现是docker pull的mongo的镜像

docker运行镜像 

systemctl restart docker
docker images
docker run --name mongo -p 27017:27017 -d mongo --auth   # 补全
docker ps -a
docker start 64ca

端口不对，处理端口问题 

cd /var/lib/docker/containers/64ca58ae81...ddfc992ac212/   # 镜像ID
vim hostconfig.json
...
"PortBindings":{"27017/tcp":[{"HostIp    ":"","HostPort":"8081"}]},

"HostPort":"8081"这里是宿主机端口
"27017/tcp"这里是容器端口

...

改完重启docker和容器

systemctl restart docker
docker stop 64
docker start 64

主机再次启动程序

2020-05-06 10:23:19.244  INFO 18145 --- [.168.1.113:8081] org.mongodb.driver.cluster               : Monitor thread successfully connected to server with description ServerDescription{address=192.168.1.113:8081, type=STANDALONE, state=CONNECTED, ok=true, version=ServerVersion{versionList=[4, 2, 1]}, minWireVersion=0, maxWireVersion=8, maxDocumentSize=16777216, logicalSessionTimeoutMinutes=30, roundTripTimeNanos=5891523}

显示mongodb连接上了,程序也正常运行😘

事后

五一放假公司测试服务器关机，上班后恢复，启动程序发现一个程序一直启动失败，调查原因才有的上文所述。

补充：

如果容器内端口从没有暴露，需要在修改config.v2.json 

vim config.v2.json

在 config.v2.json 里面添加一个配置项 

"ExposedPorts":{"27017/tcp":{}},

将这个配置项添加到 “Tty” 前面即可

最后重启docker和容器，使用docker ps验证端口映射
